## Plan to Implement Gaps in Zero Angular Lib Compared to SolidJS

### Issues Identified

1. **Missing Reactivity in Queries**: `injectQuery` returns signals that are set once but don't update when query results change.
2. **Incomplete AngularView**: The `push` method doesn't process changes, preventing incremental updates.
3. **Minimal Effect Integration**: `zero-effect.ts` has placeholder implementation.

### Changes Needed

1. **Update `inject-query.ts`**:
   - Change `data` and `details` to computed signals based on `state`.
   - Remove manual `updateSignals` call.

2. **Update `angular-view.ts`**:
   - Implement `push` method to apply changes to the entry and update the state signal.
   - Handle different change types (add, remove, etc.) to incrementally update the entry.
   - Return proper stream from `push`.

3. **Update `zero-effect.ts`**:
   - Implement proper query materialization using Effect streams.
   - Subscribe to query changes and map to Effect streams.

### Verification

- Run `npm run lint` and `npm run typecheck` to ensure no errors.
- Create a test component that uses `injectQuery` and verify signals update on data changes.
- Test Effect integration if possible.

### Critical Files

- `projects/zero-angular/src/lib/inject-query.ts`
- `projects/zero-angular/src/lib/angular-view.ts`
- `projects/zero-angular/src/lib/effect/zero-effect.ts`
